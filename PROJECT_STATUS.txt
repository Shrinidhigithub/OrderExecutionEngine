 ARCHITECTURE
══════════════════════════════════════════════════════════════════════════════

Order Submission (POST)
         ↓
    Validation
         ↓
  Store in PostgreSQL
         ↓
   Enqueue in BullMQ
         ↓ (WebSocket connects)
   Subscriber awaits updates
         ↓
    Worker picks up job
         ↓
  Fetch Raydium quote (200ms)
  Fetch Meteora quote (200ms)
         ↓
  Compare prices
         ↓
  Route to better DEX
         ↓
  Simulate execution (2-3s)
         ↓
  Update PostgreSQL
         ↓
  Publish to Redis channel
         ↓
  Subscriber receives status
         ↓
  Complete → confirmation/failure

ORDER LIFECYCLE
══════════════════════════════════════════════════════════════════════════════

pending → routing → building → submitted → confirmed/failed

Status meanings:
  • pending:   Order received and queued
  • routing:   Fetching DEX quotes from both venues
  • building:  Creating transaction
  • submitted: Transaction sent to network
  • confirmed: Success (includes txHash)
  • failed:    Error (includes error message)
